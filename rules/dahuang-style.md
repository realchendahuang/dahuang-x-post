# Dahuang Style Rules

这份规则负责中文 X 语感。

外部资料给结构，大黄规则给味道。

## 一句话核心

像一个真的做过事的人，在信息流里直接复盘。

不要像公众号。

不要像课堂。

不要像 AI 在总结方法论。

## 1. 观点前置

第一句话先亮判断。

不要先铺背景。

不要先说：

- `我最近有个感受`
- `最近很多朋友问我`
- `在当下这个时代`
- `今天想聊聊`
- `先说结论`

直接说：

- `中文 X 最浪费的一类内容，是把项目更新写成公告。`
- `我现在越来越确定，评论区不是补充，是第二条内容。`
- `发开源项目，第一句贴仓库通常是下策。`

## 2. 一行一句

一行只承载一个信息：

- 判断。
- 理由。
- 例子。
- 转折。
- 动作。

如果一行里有两个意思，拆。

如果一个段落超过 3 行，拆。

如果拆完仍然读起来像文章，重排，不要继续润色。

## 3. 短句高密度

短，不等于空。

每 2-3 行至少要有一个具体东西：

- 数字。
- 项目名。
- 时间。
- 用户原话。
- 截图说明。
- 操作步骤。
- 对比。
- 失败教训。

弱：

`这件事很重要，值得每个创作者重视。`

强：

`我只换了第一行，评论从"支持"变成了具体提问。`

## 4. 去爹味

不要教育别人。

不要写：

- `你必须`
- `你应该`
- `大家一定要`
- `只有...才能`

改成复盘：

弱：

`你必须持续输出，才能获得更多曝光。`

强：

`我后来发现，断更最伤的不是曝光，是别人忘了你在做什么。`

## 5. 去车轱辘话

同一个意思出现两次，删一次。

解释比判断还长，砍解释。

一句话没有新信息，删。

判断站不住，不要靠排比撑气势。

## 6. 禁用壳

默认删：

- `不是...而是...`
- `不只是...更是...`
- `本质上`
- `值得注意的是`
- `随着...不断发展`
- `在当下这个时代`
- `让我们一起`
- `未来可期`
- `综上所述`
- `你怎么看？`
- `欢迎在评论区留言`
- `省流版`
- `一句话总结`

替换方法：

- `不是 A 而是 B` -> 直接说 B，A 如果有价值另起一行。
- `值得注意的是 X` -> 直接说 X。
- `你怎么看` -> 问一个具体问题。

## 7. 不滥用排比

三连排比容易变 AI 味。

如果连续三行同构，打断其中一行。

弱：

```md
要持续输出。
要持续互动。
要持续复盘。
```

强：

```md
持续输出只是第一层。

更关键的是：
别人能不能反复看到你在同一个方向上出现。
```

## 8. X 信息流排版

读者不是坐下来读文章。

读者在扫。

所以：

- 第一行抓判断。
- 第二行给代价或结果。
- 第三行给具体证据。
- 中间少解释，多推进。
- 结尾给动作或余味。

不要把完整逻辑一次塞满。

主帖先把人留住。

评论区再补材料。

出来混，最重要的是出来。

不要内耗太久。

先发出来。

持续露脸，比偶尔憋一条大稿有用。

## 9. 评论区承接

主帖抓人。

首评补资料。

评论区继续营业。

适合放评论区的东西：

- 链接。
- 仓库。
- Demo。
- 截图。
- 更长解释。
- 新证据。
- 用户问题的延伸回答。

如果主帖有外链，先问自己：

`这条去掉链接后，还值不值得看？`

不值得，就先改主帖。

## 10. 爆帖二次营销

爆帖不是终点，是入口。

具体怎么拆、怎么二次承接、怎么把新帖贴回原帖评论区，见 `references/distribution-loop.md` 的「爆帖二次利用」。

原则只有一句：

不要复读原帖。

每次复用都要换读者、场景、动作、证据。

## 11. 个人品牌转化

X 内容不只涨粉。

它也可以导向：

- GitHub star。
- 产品验证。
- Newsletter。
- 用户线索。
- 项目曝光。
- 招聘合作。
- 真实反馈。

但主帖别像广告。

先让判断成立。

再让链接出现。

## 12. 中文 X 语感

可以直接。

可以有点怪。

可以锋利。

可以说人话。

但不要为了显得会写，牺牲信息密度。

最终读起来要像：

`这个人真的做过，真的踩过坑，现在把最有用的一句拎出来了。`
