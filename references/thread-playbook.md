# Thread Playbook

Thread 不是长文切片。

Thread 是一组能独立传播、又能连续推进的短帖。

## 1. 什么时候写 Thread

适合写 Thread：

- 有 3 个以上独立观点。
- 有完整故事线：起因、动作、转折、结果。
- 有步骤教程。
- 有证据链。
- 有截图/图表/Demo。

不适合写 Thread：

- 只有一个判断。
- 只是想把废话装下。
- 每条都必须靠上下文才看懂。

只有一个判断时，写短帖 + 首评。

## 2. Lead-in 第一条

第一条像电影预告片。

它要在 1-3 句里说明：

- 给谁看。
- 讲什么。
- 为什么值得点开。
- 读完得到什么。

模板：

```md
{强判断/结果/冲突}

{为什么多数人会错过}
{这条 Thread 会给什么}
```

例：

```md
很多开源项目没人看，不是项目不够好。

是发布帖写成了 README 摘要。

我把一个项目帖拆成 6 条，结构如下：
```

## 3. 主体写法

每条只讲一个 idea。

每条都要能单独被转发。

每条结构优先：

```md
{小判断}

{例子/证据/动作}
```

不要写成：

```md
第一，...
第二，...
第三，...
```

可以写成：

```md
1/
第一条先别讲功能。

先讲"以前多麻烦，现在少了哪一步"。
```

## 4. 7 种 Thread 类型

### Advice Thread

适合经验建议。

```md
1/ X 文案别先写标题。
先写判断。

2/ 判断不够硬，标题再花也没用。

3/ 判断够硬，第一句直接发都能成立。
```

### Complex Topic Thread

适合复杂概念。

```md
1/ "链接放首评"不是迷信。

2/ 它背后是一个更简单的原则：
主帖先留住人，外链后置。
```

### Expert Skill Thread

适合展示专业方法。

```md
1/ 我改 X 文案只看 4 个位置：

第一行、第三行、首评、最后一句。
```

### How I Did It

适合复盘。

```md
1/ 我把一个普通项目更新帖改成了 3 条内容。

一条讲结果。
一条讲坑。
一条讲技术细节。
```

### Story Thread

适合个人经历。

```md
1/ 这个标题本来差点发出去了。

用户的一句话救了它。
```

### Visual Thread

适合截图、图表、Demo。

```md
1/ 先看这张图。

改前：用户要点 5 次。
改后：只剩 1 次。
```

### Compilation Thread

适合资源合集。

```md
1/ 我最近收藏了 7 个适合独立开发者的 X 写法。

不是模板。
是能直接改项目更新的结构。
```

## 5. TL;DR 先写

写 Thread 前先写最后的 TL;DR。

如果总结不出来，说明主线不清楚。

TL;DR 可以变成：

- Thread 大纲。
- 最后一条 recap。
- 主帖短版。
- Newsletter 摘要。

## 6. CTA

CTA 不是一定要关注。

根据目标选：

- GitHub：`仓库和 Demo 放首评。`
- 产品验证：`我想找 5 个真实用户试一下。`
- Newsletter：`长版我会整理到 Newsletter。`
- 互动：`你发项目更新时，最难写的是第一句还是首评？`
- 复用：`这个主题我后面会拆 3 条。`

不要写空 CTA：

`喜欢请关注。`

## 7. 发之前过一遍

逐条问：

- 第一条不展开也能成立吗？
- 前 3 句有没有 who / what / why？
- 每条是否只有一个 idea？
- 有没有一条像大段文章切片？
- 最后一条是否给了下一步？
- 链接是否放到最后或首评，没压在第一条？
