# Viral Post Anatomy

判断一条中文 X 帖有没有爆款潜质。

和 `viral-post-playbook.md` 的分工：

- 本文件：判断潜质（这条命中了爆帖的哪些特征）。
- playbook：怎么改（Hook 怎么起、正文怎么排）。

## 一句话判断

它有没有把一个很多人隐约有感受、但没说清楚的问题，用一句明确的判断说出来。

有，就有潜质。没有，润色也救不回来。

## 推荐机制在奖励什么

X 的 For You 从你关注和没关注的人里选内容，按 Likes、Reposts、Replies、active minutes 等互动信号排序，也会推 popular 的、和你圈子互动过的内容。([X 推荐系统说明](https://help.x.com/en/resources/recommender-systems/for-you-home-timeline-recommendations))

一条帖要爆，核心是让第一批人愿意停、点赞、回复、转发。写得好只是前提，让人愿意互动才是杠杆。

Bear Liu 的 X 增长复盘里提到：情绪内容易转发，价值内容易点赞；简中 X 里 AI、科技、英语方向互动密度高；爆了之后人会点进主页，档案页接不住就白来。([beartalking.com](https://beartalking.com/x-growth))

## 爆款潜质 7 特征

判断一条帖时挨个对照，命中越多越值得打磨。

### 1. 观点前置

开头不铺垫。前 1 秒读者只判断一件事：这人想说什么。

弱：`最近我在思考一个问题……`

强：

- `独立开发者最大的机会，不在模型，而在基础设施。`
- `Cloudflare 对普通人的价值，是把部署变便宜、变轻、变傻瓜。`
- `AI Coding 真正拉开差距的地方，是产品判断，不是会不会写 Prompt。`

开头直接给判断，别人哪怕不同意，也知道怎么回。怎么打开头见 `viral-post-playbook.md` 的 Hook 矩阵。

### 2. 中文 X 共同痛点

容易爆的不是纯知识，是"我也遇到过"的东西。

好痛点同时满足三个条件：人群够大、痛感够强、讨论成本低。

例：

- AI Coding 翻车。
- Cloudflare 看不懂。
- Codex / Claude Code 怎么选。
- 独立开发者没钱买服务器。
- 普通人想做产品，不知道从哪开始。
- 大厂流程太重，小团队反而跑得快。

### 3. 可转发价值

点赞是认可，转发是帮自己立人设。

一条帖能不能爆，看别人转发它时会不会觉得自己变懂行。

自带转发价值的形态：

- 一个判断。
- 一个清单。
- 一个避坑。
- 一个对比。
- 一个资源。
- 一个能截图的结论。

例：写 Cloudflare，最容易转发的不是"Cloudflare 很好"，而是：

`独立开发者用 Cloudflare，最该先搞懂这 6 个东西：Workers、Pages、R2、D1、Hyperdrive、AI Gateway。`

收藏价值（数字/判断/动作/金句/信息增量）见 `viral-post-playbook.md` 的「可收藏价值」。

### 4. 轻微冲突，不是骂人

中文 X 吃冲突。最好的冲突是判断有棱角，不是情绪宣泄。

例：

- `很多 AI Coding 教程讲得太虚，普通人真正卡住的是部署、数据库、鉴权、支付、日志。`
- `别一上来研究微服务。独立开发者先把一个产品跑通，比架构优雅重要。`
- `技术选型最怕装逼。MVP 阶段，便宜、稳定、能上线，优先级最高。`

这种冲突有用：在帮用户做判断，不是单纯喷。

### 5. 留评论入口

推荐机制看 Replies 这类正向信号。只适合被看完划走的帖，写得再好也不一定爆。

别问假的"你怎么看？"。问具体的：

- `这套技术栈你会怎么改？`
- `你觉得 Cloudflare 最坑的是哪个服务？`
- `你用 Codex 最大的卡点是什么？`
- `还有哪个免费工具值得放进独立开发者工具箱？`

评论区一活，帖子更容易继续扩散。

### 6. 适合信息流

不是大段文章，是一块一块的判断。一行一句。短。硬。能扫读。每三五行一个信息点。

怎么排见 `viral-post-playbook.md` 的「爆帖化正文结构」和 `dahuang-style.md`。

### 7. 身份标签

中文 X 转发常带身份认同。让某一类人心里说"对，说的就是我"，就有爆款基础。

把对象说清楚：

- `给独立开发者的 Cloudflare 入门路线。`
- `给普通人跑长期 AI 工作流的低成本方案。`
- `给 AI Coding 新手的后端常识清单。`
- `给想做一人公司的人的工具栈。`

这类句子天然比泛泛而谈强。

## 注意：爆款 ≠ 涨粉

这不是"命中越多越好"的特征，是一个要警惕的坑。

有些帖会爆但不涨粉：热点、骂战、情绪站队、八卦。

有些帖浏览没那么夸张但涨粉强：高质量教程、项目复盘、工具清单、个人经验、开源仓库。

判断一条帖不只看会不会爆，还要看：爆了之后来的粉是不是我要的人。

爆了之后人会点进主页——档案页、自我介绍、头像、链接页接不住，流量就白来。

## 判断公式

爆款潜质 = 大众痛点 × 明确判断 × 可转发价值 × 评论入口 × 人设匹配

少一个都能发。全都有，就值得重点打磨。

## 最稳的爆款结构

观点型开头 + 干货型正文 + 项目/仓库承接 + 评论区继续补充。

```md
普通人想做 AI 产品，最该补的不是模型知识，是基础设施常识。

因为产品真正上线后，卡你的是：

鉴权
数据库
部署
存储
日志
限流
支付
成本控制

我现在越来越觉得，Cloudflare 是独立开发者最值得研究的一套基础设施。

便宜，够用，闭环快。

我整理了一个 Cloudflare Playbook，专门给 AI Coding / 独立开发者用。
```

有判断。有痛点。有清单。有身份标签。有项目承接。有转发理由。
