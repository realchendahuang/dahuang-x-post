# Viral Post Playbook

这份文件把外部资料沉淀成可执行的短帖改写方法。

目标不是保证爆。

目标是让帖子具备爆帖该有的形态：

- 第一眼能停住。
- 三行内能懂收益。
- 内容有信息增量。
- 读者有话可回。
- 后续能被评论区和新帖继续承接。

## 1. Hook 的两个任务

第一句只有两个任务：

1. 让读者停下。
2. 让读者继续看。

如果一个词不服务这两个任务，删。

弱：

`最近我在思考 X 内容运营这件事。`

强：

`中文 X 最浪费的一类内容，是把项目更新写成公告。`

为什么强：

- 有明确对象：中文 X。
- 有判断：项目更新不该写公告。
- 有轻微冲突：很多人就是这么写的。

## 2. 前 3 行必须交代 who / what / why

这来自 Thread hook 和 headline 资料里的共同原则：读者要快速知道"给谁、讲什么、为什么值得看"。

模板：

```md
{给谁的判断}

{这件事为什么反常识/有代价}
{读完能得到什么}
```

例：

```md
独立开发者发 X，最吃亏的是把更新写成 release note。

别人看到的是"你又发版了"。
但真正能传播的，是"这个版本解决了谁的具体问题"。
```

## 3. 可传播判断

Hook 承载的判断要硬。

硬判断至少满足 2 个：

- 有反差：多数人会想反。
- 有结果：做了之后发生了什么。
- 有代价：不这么做会损失什么。
- 有细节：数字、时间、截图、动作、项目名。
- 有身份：让目标读者感觉"这说的是我"。

一个条件都不沾的判断，再短也传不出去。

例：`我最近对 X 运营有一些思考。` —— 没反差、没结果、没身份。

换成：`中文 X 最浪费的一类内容，是把项目更新写成公告。` —— 有对象、有判断、有冲突。

## 4. Hook 矩阵

同一个素材至少先打 5 个开头，再选。

### A. 直给判断

适合观点、经验、复盘。

```md
中文 X 不缺教程，缺真实复盘。
```

### B. 结果感

适合产品、增长、实验。

```md
这个改法有点猛。

我只换了第一行，回复质量直接变了。
```

### C. 反常识

适合让人回复和 Quote。

```md
链接不是不能发。

问题是很多人把链接当主角。
```

### D. 精准读者

适合技术、独立开发、AI 工具。

```md
如果你发的是 GitHub 项目，别第一句就贴仓库。
```

### E. 具体数字

适合有真实数据时。

```md
我把一个 8 分钟的流程压到 20 秒。

但这条帖不能写成"我做了个工具"。
```

### F. 失败/踩坑

适合复盘。

```md
我之前发项目帖最大的错：

把自己做了什么讲清楚了。
但没讲别人为什么要关心。
```

### G. 故事瞬间

适合个人经历。

```md
昨天有个用户的一句话，把我原来的标题推翻了。
```

### H. 告白/承认

适合建立信任。

```md
我以前也以为 X 文案就是写短一点。

后来发现不是。
它是把判断提前，把解释后置。
```

## 5. 爆帖化正文结构

短帖不是把文章缩短，是重排信息顺序：判断提前，铺垫后置。

默认短帖结构（一行一句，每行只推进一个信息）：

```md
{第一行：判断/结果/冲突}

{第二行：为什么这事值得看}
{第三行：一个具体证据、数字、动作或场景}

{第四行：转折/经验/反直觉}
{第五行：下一步动作或评论区承接}
```

删掉所有铺垫型开头。

按内容选一种正文形态：

### 判断型

```md
{判断}

{为什么多数人会做错}
{一个具体例子}

{你的经验}
{轻 CTA 或首评承接}
```

### 结果型

```md
{结果}

{你做了什么}
{为什么这一步有效}

{可复制动作}
{首评放证据/链接}
```

### 争议型

```md
{反常识判断}

{先承认对方为什么合理}
{再指出真正问题}

{给一个边界}
{问一个具体问题}
```

### 工具/项目型

```md
{它把什么痛点变简单了}

{原来怎么麻烦}
{现在怎么做}

{适合谁}
{链接/仓库放首评}
```

## 6. 可收藏价值

一条帖想被收藏，至少要有两个：

- 一个具体数字。
- 一个独特判断。
- 一个可操作动作。
- 一个能复述的金句。
- 一个读完才知道的信息增量。

弱：

`持续输出很重要。`

强：

`断更最伤的不是曝光，是别人忘了你在做什么。`

## 7. 常见坏稿改法

### 公告腔

弱：

```md
我们发布了新版本，新增了多项功能，欢迎大家体验。
```

改：

```md
这个版本最值得看的不是功能多。

是我把新用户第一次上手的步骤砍掉了 3 个。
```

### 公众号腔

弱：

```md
在当今快速变化的 AI 时代，内容创作者需要重新思考自己的表达方式。
```

改：

```md
AI 时代的内容问题，不是写不出来。

是太多人写出来以后，像没活过。
```

### 课堂腔

弱：

```md
你需要做到以下三点，才能提升 X 账号影响力。
```

改：

```md
我后来发现，X 影响力不是靠一条爆帖撑起来的。

是靠别人反复看到你在同一个方向上出现。
```

### 硬互动

弱：

```md
你怎么看？欢迎评论区留言。
```

改：

```md
你发项目更新时，链接一般放主帖还是首评？
```

## 8. 交付前删干净

发之前扫一遍，能删的删：

- 自我铺垫：`我最近有个感受`
- 大词背景：`在当今时代`
- 论文转场：`值得注意的是`
- AI 套壳：`不是...而是...`
- 伪总结：`综上所述`
- 假互动：`你怎么看`
- 空 CTA：`欢迎关注`

删不掉的，换成具体判断、具体问题、具体动作。
