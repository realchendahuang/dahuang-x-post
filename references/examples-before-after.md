# Before And After Examples

这些例子展示这个 skill 怎么改，别照抄。

## 1. 项目公告改成 X 主帖

原稿：

```md
我们今天正式发布了 dahuang-x-post，这是一个面向中文创作者的 X 文案优化 skill。它可以帮助用户优化推文、生成 Thread、改写开头，并提升内容在 X 平台的表现。欢迎大家试用并提出反馈。
```

问题：

- 第一眼是公告。
- 没有具体痛点。
- "提升表现"太空。
- 链接和试用 CTA 太早。

改后：

```md
中文 X 文案最常见的问题：

内容是好的。
第一句是废的。

我做了个 skill，专门干这件事：
把公众号腔、公告腔、AI 腔，改成能在信息流里停住人的帖子。

仓库放首评。
```

首评：

```md
仓库：
{url}

它默认会给主帖、首评和后续复用角度。
不是润色，是重排信息顺序。
```

## 2. 公众号开头改成 X 短帖

原稿：

```md
在当今快速发展的 AI 时代，内容创作者面临着前所未有的机遇和挑战。如何在信息爆炸的环境中抓住用户注意力，已经成为每个创作者必须认真思考的问题。
```

改后：

```md
AI 时代最尴尬的内容问题：

大家都能写了。
但很多人写出来以后，像没活过。

真正稀缺的不是文字。
是判断、现场感和具体经历。
```

## 3. 技术项目改成结果感

原稿：

```md
我做了一个命令行工具，可以自动分析项目中的无用文件，并生成清理建议。它支持多种文件类型，使用简单，欢迎试用。
```

改后：

```md
我把项目清理这件事，从半小时压到了 30 秒。

以前要人工翻目录。
现在跑一个命令，直接列出可疑无用文件和原因。

最爽的不是自动删除。
是它先让你看懂为什么能删。

Demo 放首评。
```

## 4. 爹味建议改成复盘口吻

原稿：

```md
想要做好 X 运营，你必须坚持每天发帖，并且多参与评论区互动。只有持续输出高质量内容，才能获得更多曝光。
```

改后：

```md
我后来发现，X 断更最伤的不是曝光。

是别人忘了你在做什么。

每天发一条不一定会爆。
但持续出现，会让同一批人反复记住你的方向。

这比偶尔憋一条大稿靠谱。
```

## 5. 长文拆 Thread

原稿主题：

```md
如何把一个 GitHub 开源项目发布到 X，并获得更多 star。
```

Thread：

```md
1/
开源项目发 X，别第一句就贴 GitHub。

陌生人不关心仓库。
他们先关心：这个东西帮我少做了哪一步。

2/
第一条讲结果。

弱：我做了一个 CLI 工具。
强：我把项目清理从半小时压到 30 秒。

3/
第二条讲旧痛点。

以前要人工翻目录、猜哪些文件能删。
这一步最烦，也最容易删错。

4/
第三条讲新动作。

现在跑一个命令。
它列出可疑文件，还解释为什么可能无用。

5/
第四条放证据。

截图、Demo、前后对比，都比抽象功能点更有用。

6/
最后再放仓库。

人已经知道它解决什么问题了。
这时候链接才有意义。
```

## 6. 泛泛提问改成可回复问题

弱：

```md
你怎么看？
```

强：

```md
你发项目更新时，第一句通常写结果，还是先写自己做了什么？
```

为什么强：

- 问题具体。
- 和主帖动作相关。
- 回复成本低。
- 能引出真实经验。
